arguments

arguments是JavaScript免费提供的一个关键字，它只在函数内部起作用，并且永远指向当前函数的调用者传入的所有参数。argumentss类似数组但不是数组。

全局作用域

不在任何函数内定义的变量就具有全局作用域。JavaScript默认有一个全局对象window，全局作用域的变量实际上呗绑定到window的一个属性。
JavaScript的函数定义有个特点，它会先扫描整个函数体的语句，把所有声明的变量“提升”到函数顶部

##this
在一个函数内部，this是一个特殊变量，它始终指向当前对象，所以通过this.可以拿到当前对象的属性

this指针只在age方法的函数内指向obj对象，而在函数内部定义的函数，this又指向了undefined(在非strict模式下，它重新指向全局对象window)
解决方案：使用apply或者call来重新定义this的指向
要指定函数的this指向哪个对象，可以用函数本身的apply方法，它接收两个参数，第一个参数就是需要绑定的this变量，第二个参数是数组，表示函数本身的参数。

嵌套的函数不会从调用它的函数中继承this。如果嵌套函数作为方法调用，其this的值指向调用他的函数。如果嵌套函数作为函数调用，其this值不是全局对象就是undefined。如果你想访问这个外部函数的this值，需要将this的值保存在一个变量里，这个变量和内部函数在一个作用域内。

当访问一个对象的属性时，JavaScript引擎首先在该对象上查找该属性，如果没有找到，就会去其原型上找，如果还没有找到，就一直上溯到Object.prototype对象，最好还没有找到，返回undefined。

JavaScript继承实现

首先定义一个父类

1、原型链继承

核心：将父类的实例作为子类的原型

特点：
1、非常纯粹的继承关系，实例是子类的实例，也是父类的实例
2、父类新增原型方法/原型属性，子类都能访问到
3、简单、易于实现
缺点：
1、想要为子类新增属性和方法，必须要在new Animal()这样的语句之后执行，不能放到构造器中
2、无法实现多继承
3、来自原型对象的引用属性是所以实例共享的
4、创建子类实例时，无法向父类构造函数穿参

2、构造继承

核心：使用父类的构造函数来增强子类实例，等于是复制父类的实例属性给子类（没用到原型）

特点：
1、解决了1中，子类实例共享父类引用属性的问题
2、创建子类实例时，可以向父类传递参数
3、可以实现多继承（call多个父类对象）
缺点：
1、实例并不是父类的实例，只是子类的实例
2、只能继承父类的实例属性和方法，不能继承原型属性/方法
3、无法实现函数复用，每个子类都有父类实例函数的副本，影响性能

3、实例继承

核心：为父类实例条件新特性，作为子类实例返回

特点：
1、不限调用方式，不管是new子类（）还是子类（），返回的对象具有相同的效果
缺点：
1、实例是父类的实例，不是子类的实例
2、不支持多继承

4、拷贝继承

特点：
1、支持多继承
缺点：
1、效率较低、内存占用高（因为要拷贝父类的属性）
2、无法获取父类不可枚举的方法（不可枚举方法，不能使用for in访问到）

5、组合继承

核心：通过调用父类构造，继承父类的属性并保留传参的优点，然后通过将父类实例作为子类原型，实现函数复用

特点：
1、弥补了构造继承的缺陷，可以继承实例属性/方法，也可以继承原型属性/方法
2、即是子类的实例，也是父类的实例
3、不存在引用属性共享问题
4、可传参
5、函数可复用
缺点：
1、调用了两次父类构造函数，生成了两份实例（子类实例将子类原型上的那份屏蔽了）

6、寄生组合继承

核心：通过寄生方式，砍掉父类实例属性，这样，在调用两次父类的构造的时候，就不会初始化两次实例方法/属性，避免组合继承的缺点

特点：
1、看成完美
缺点：
1、实现较为复杂

Webpack属性总结

webpack 简介

webpack是一个模块打包工具。它使得模块相互依赖并且可构建等价于这些模块的静态资源。相比于已经存在的模块打包器，webpack的开发动机是实现代码分包和通过模块化完成代码的威风集成。webpack可以根据项目需求合并代码，并且支持按需加载。
webpack的实现目标是：
1.拆分依赖树为多个按需加载的chunk
2.保证快速首屏加载
3.每种静态资源都可以成为模块
4.能够将第三方库视作一个模块来处理
5.能够制定模块打包器的几乎任何部分
6.适合大型项目

基本属性

entry

entry英文翻译为入口，其实就是模块的入口
1.entry的值是字符串，这个字符串对应的模块会在启动的时候加载
2.entry的值是数组，这个数组内所有模块会在启动的时候加载，数组的最后一个元素作为export
3.entry的值是对象，可以构建多个bundle

output

对应entry的输出规则，通过output对象来实现

path: 指定输出文件路径
filename: 输出文件名
publicPath: 设置为想要的资源访问路径。访问时，则需要通过类似http://localhost:8080/asstes/index.js来访问资源，如果没有设置，则默认从站点根目录加载

loader

loader是wenbpack中比较重要的部分，是处理各类资源的执行者。是一系列的函数，将资源中的代码作为参数，然后返回新的代码。你可以用loader告诉webpack可以加载哪些文件，或者不加载哪些文件。
Loader的特点
1.可以链式执行。它们在一个管道中被提交，只需要保证最后的loader返回JavaScript即可，其他loader可以返回任意方便下一个loader处理的内容。
2.可以异步或者同步执行
3.运行在Node.js中，可以做几乎任何事儿
4.可以接收query参数，用于向loader传递参数
5.配置中可与正则/扩展结合使用
6.可以在npm中发布并使用
7.除了main,其他模块可以导出成loader
8.可以通过配置调入
9.和插件（plugins）配合可获得更多功能
10.可生成其他格式文件

loader支持多种类型

常用的xxx-loader类型有：html、css、jsx、coffee、jade、less、sass、style

配置loader

test: 匹配希望处理文件的路径
exclude: 匹配不希望处理文件的路径
loaders: xxx-loader可以简写为xxx，？后以query方式传递给loader参数

处理图片、字体等文件

在css中或者js做集中，都会涉及到require图片的情况，webpack可以内联图片地址到打包js中并且通过require()返回图片路径。当然,不只是图片，还有css种用到的iconfont。特殊情况用到的flash等，都可以相似处理。这里，需要用到url-loader或file-loader。

Redux

三大原则

单一数据原则

整个应用的state被储存在一棵object tree中，并且这个object tree只存在于唯一一个store中

state是只读的

action是一个用于描述已发生事件的普通对象。视图和网络请求都不能直接修改state，它们只能表达想要修改的意图。所有的修改都被集中化处理，且严格按照一个接一个的顺序执行,因此不用担心race condition 的出现。 Action是普通对象,可以被日志打印,后期调试或测试时回放出来

使用纯函数来执行修改

为了描述action如何改变state tree ，需要编写reducers,reducer只是一些纯函数它接收先前的state和action，并返回新的state。刚开始可以只有一个reducer，随着应用变大，可以拆成多个小的reducers，分别独立地操作state tree 的不同部分,甚至编写可复用的reducer来处理一些通用任务，如分页器

基础点

Action

Action是传输数据的载体，一般我们会用到dispatch这个运输工具来运输数据。Action本质上是一个JavaScript普通对象，在action内必须使用使用一个字符串类型的type字段来表示将要执行的动作，在一般情况下type会定义成一个常量，当大规模使用type时应该使用一个单独的模块来存放action

Reducer

Action用来描述将要执行的操作，没有真正的去更新state，需要reducer来执行这一操作。Reducer是一个纯函数，接受就得state和action，返回新的state

Js中的数据类型

js中的数据类型

undefined null boolean string number object

Undefined类型

Undefined类型只有一个值，即特殊的undefined。声明完一个变量但为对其加以初始化时，这个变量的值就是undefined,当声明一个变量时未经初始化的值就会默认取得undefined值。
var message；
typeof message // undefined
message == undefined true

Null类型

Null类型也是只有一个值，这个值就是null。从逻辑的角度来讲，null值表示一个空对象指针。当使用typeof来判断类型时，就会出现typeof null 为object的情况。Null的使用，当定义一个变量时，如果准备将来用于保存对象，那么最好将该变量初始化为null而不是其他值。
Undefined和Null的关系
事实上，undefined值是由null派生而来的，因此判断他们的的相等性时会返回true，即alert(undefined == null); 返回true。

Boolean类型

Boolean类型是比较常用的类型，该类型只有两个字面值true和false。要注意的一点是，这两个值与数字值不是一回事，true不一定等于1，而false也不一定等于0.true和false是区分大小写的。

Number类型

Number类型，这里主要说一下一个特殊的数值，NaN即非数值（Not a Number），这个数值用于表示一个本来要返回数值的操作是未返回数值的情况。3个可以将非字符型的数值转化为字符型：Number()、paeseInt()、parseFloat()。

String类型

String类型用于表示字符串，字符串可以有双引号或单引号表示。
字符串的特点：
ECMAScript中的字符串是不可以改变的，字符串一旦被创建，他们的值就不会改变。要改变某个变量保存的字符串，首先要销毁原来的字符串，然后再用另一个新的包含新值得字符串填充该变量。
转化为字符串：
使用几乎每个值都有的toString()方法。
