Webpack属性总结

发表于 2016-09-24

webpack 简介

webpack是一个模块打包工具。它使得模块相互依赖并且可构建等价于这些模块的静态资源。相比于已经存在的模块打包器，webpack的开发动机是实现代码分包和通过模块化完成代码的威风集成。webpack可以根据项目需求合并代码，并且支持按需加载。
webpack的实现目标是：
1.拆分依赖树为多个按需加载的chunk
2.保证快速首屏加载
3.每种静态资源都可以成为模块
4.能够将第三方库视作一个模块来处理
5.能够制定模块打包器的几乎任何部分
6.适合大型项目

基本属性

entry

entry英文翻译为入口，其实就是模块的入口
1.entry的值是字符串，这个字符串对应的模块会在启动的时候加载
2.entry的值是数组，这个数组内所有模块会在启动的时候加载，数组的最后一个元素作为export
3.entry的值是对象，可以构建多个bundle

output

对应entry的输出规则，通过output对象来实现

path: 指定输出文件路径
filename: 输出文件名
publicPath: 设置为想要的资源访问路径。访问时，则需要通过类似http://localhost:8080/asstes/index.js来访问资源，如果没有设置，则默认从站点根目录加载

loader

loader是wenbpack中比较重要的部分，是处理各类资源的执行者。是一系列的函数，将资源中的代码作为参数，然后返回新的代码。你可以用loader告诉webpack可以加载哪些文件，或者不加载哪些文件。
Loader的特点
1.可以链式执行。它们在一个管道中被提交，只需要保证最后的loader返回JavaScript即可，其他loader可以返回任意方便下一个loader处理的内容。
2.可以异步或者同步执行
3.运行在Node.js中，可以做几乎任何事儿
4.可以接收query参数，用于向loader传递参数
5.配置中可与正则/扩展结合使用
6.可以在npm中发布并使用
7.除了main,其他模块可以导出成loader
8.可以通过配置调入
9.和插件（plugins）配合可获得更多功能
10.可生成其他格式文件

loader支持多种类型

常用的xxx-loader类型有：html、css、jsx、coffee、jade、less、sass、style

配置loader

test: 匹配希望处理文件的路径
exclude: 匹配不希望处理文件的路径
loaders: xxx-loader可以简写为xxx，？后以query方式传递给loader参数

处理图片、字体等文件

在css中或者js做集中，都会涉及到require图片的情况，webpack可以内联图片地址到打包js中并且通过require()返回图片路径。当然,不只是图片，还有css种用到的iconfont。特殊情况用到的flash等，都可以相似处理。这里，需要用到url-loader或file-loader。

三大原则

单一数据原则

整个应用的state被储存在一棵object tree中，并且这个object tree只存在于唯一一个store中

state是只读的

action是一个用于描述已发生事件的普通对象。视图和网络请求都不能直接修改state，它们只能表达想要修改的意图。所有的修改都被集中化处理，且严格按照一个接一个的顺序执行,因此不用担心race condition 的出现。 Action是普通对象,可以被日志打印,后期调试或测试时回放出来

使用纯函数来执行修改

为了描述action如何改变state tree ，需要编写reducers,reducer只是一些纯函数它接收先前的state和action，并返回新的state。刚开始可以只有一个reducer，随着应用变大，可以拆成多个小的reducers，分别独立地操作state tree 的不同部分,甚至编写可复用的reducer来处理一些通用任务，如分页器

基础点

Action

Action是传输数据的载体，一般我们会用到dispatch这个运输工具来运输数据。Action本质上是一个JavaScript普通对象，在action内必须使用使用一个字符串类型的type字段来表示将要执行的动作，在一般情况下type会定义成一个常量，当大规模使用type时应该使用一个单独的模块来存放action

Reducer

Action用来描述将要执行的操作，没有真正的去更新state，需要reducer来执行这一操作。Reducer是一个纯函数，接受就得state和action，返回新的state
